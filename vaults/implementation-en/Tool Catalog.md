# Tool Catalog

The complete list of all [[MCP Primitives|tools]] available across all servers, auto-discovered by the [[prompt-engineer]] at startup. It is the [[Ontology as Code|ontology]] of the system's capabilities -- the model knows what it can and cannot do.

---

## Architecture

[[Marvin]] is a unified [[FastMCP]] server. The catalog is a Python list in `marvin_server.py`:

```python
MARVIN_TOOLS = [
    "retrieve", "get_concept", "traverse", "why_exists", "list_concepts",
    "set_aliases", "batch_set_aliases",
    "log_decision", "log_session",
    "expand", "link", "auto_link", "ensure_bidirectional",
    "propose_schema_change", "execute_schema_change",
    "search_docs", "list_docs", "get_doc",
    "fetch_url", "save_doc", "rank_urls", "crawl_docs", "research_topic",
    "generate_prompt", "refine_prompt", "audit_prompt",
    "generate_diagram", "judge_diagram", "save_diagram", "list_diagrams", "get_diagram",
    "inspect_schemas", "stats",
]
```

Each tool is a function decorated with `@mcp.tool()` in `marvin_server.py`, delegating to one of 6 backend modules. The catalog is injected into `generate_prompt` and `refine_prompt` so that every generated prompt includes the full tool list.

## The Catalog as Ontology

The catalog contains, for each tool: name, parameters (name + type), and description. Formatted in markdown:

```
### docs-server
- `search_docs(query: string)` -- Search across all documentation files...
- `list_docs()` -- List all available documentation files
- `get_doc_summary(filename: string)` -- Return the first section...

### web-to-docs
- `convert_url(url: string)` -- Fetch a webpage and return...
...
```

This catalog is the answer to "what can the system do?" It is the [[Ontology]] of capabilities -- not of the domain, but of the tools that access the domain.

## Injection into Prompts

The catalog is injected in `generate_prompt` and `refine_prompt`:

```python
f"## Available MCP Tools\n"
f"The following tools are available in this workspace. Use them in section 2 "
f"(KNOWLEDGE BEYOND WEIGHTS) of the generated prompt.\n\n"
f"{MCP_TOOL_CATALOG}\n\n"
```

Every prompt generated by the [[Transformer-Driven Prompt Architect]] includes the complete tool list. The model receiving the prompt knows exactly which tools to invoke -- no guessing needed. It is [[Anti-Hallucination]] of capabilities.

## Role in the Thesis

In the thesis, [[Tool|tools]] are [[Subset|subsets]] of [[Vector|vectors]] that function as [[Tool as Bias|bias]]. The catalog is the meta-bias -- it informs the model about all available biases. It is [[Context]] about [[Context]].

The fact that it is auto-discovered (not hardcoded) means that adding a new server with new tools automatically expands the catalog. The [[Ontology as Code|ontology]] grows without editing the prompt-engineer.
