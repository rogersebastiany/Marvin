# Tool Catalog

The complete list of all [[Primitivas MCP|tools]] available across all servers, auto-discovered by the [[prompt-engineer]] at startup. It is the [[Ontologia como Código|ontology]] of the system's capabilities -- the model knows what it can and cannot do.

---

## Auto-Discovery

The `prompt_engineer_server.py` imports the 3 sibling servers and lists their tools:

```python
from server import mcp as docs_mcp
from web_to_docs_server import mcp as web_mcp
from system_design_server import mcp as design_mcp

servers = [
    ("docs-server", docs_mcp),
    ("web-to-docs", web_mcp),
    ("system-design", design_mcp),
]

async def _gather():
    catalog = []
    for server_name, server_mcp in servers:
        tools = await server_mcp.list_tools()
        # formats each tool: name, params, description
```

`asyncio.run()` executes the synchronous discovery at import time. The result is stored in `MCP_TOOL_CATALOG` -- a global constant used in all prompt-engineer tools.

## The Catalog as Ontology

The catalog contains, for each tool: name, parameters (name + type), and description. Formatted in markdown:

```
### docs-server
- `search_docs(query: string)` -- Search across all documentation files...
- `list_docs()` -- List all available documentation files
- `get_doc_summary(filename: string)` -- Return the first section...

### web-to-docs
- `convert_url(url: string)` -- Fetch a webpage and return...
...
```

This catalog is the answer to "what can the system do?" It is the [[Ontologia]] of capabilities -- not of the domain, but of the tools that access the domain.

## Injection into Prompts

The catalog is injected in `generate_prompt` and `refine_prompt`:

```python
f"## Available MCP Tools\n"
f"The following tools are available in this workspace. Use them in section 2 "
f"(KNOWLEDGE BEYOND WEIGHTS) of the generated prompt.\n\n"
f"{MCP_TOOL_CATALOG}\n\n"
```

Every prompt generated by the [[Transformer-Driven Prompt Architect]] includes the complete tool list. The model receiving the prompt knows exactly which tools to invoke -- no guessing needed. It is [[Anti-Alucinação]] of capabilities.

## Role in the Thesis

In the thesis, [[Tool|tools]] are [[Subconjunto|subsets]] of [[Vetor|vectors]] that function as [[Tool como Bias|bias]]. The catalog is the meta-bias -- it informs the model about all available biases. It is [[Contexto]] about [[Contexto]].

The fact that it is auto-discovered (not hardcoded) means that adding a new server with new tools automatically expands the catalog. The [[Ontologia como Código|ontology]] grows without editing the prompt-engineer.
