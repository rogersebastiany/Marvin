# Anti-Hallucination

The set of mechanisms in the POC that prevent [[Hallucination]] -- the model operating outside the [[Subset]] defined by the [[Programmatic Context|context]], producing fabricated responses.

---

## The Problem

Without [[Ontology as Code|ontology]], the model operates in the entire [[Sample Space]] S. The probability of producing a factually correct response is low -- it is in the zone S \ A, the complement of [[Context]].

"If you demand something that was not previously mapped, it will hallucinate, and this happens 100% of the time."

## Mechanisms in the POC

**1. Tools as delimiters**
Each [[MCP Primitives|tool]] constrains what the agent can do. `search_docs` only searches in `docs/`. `get_diagram` only reads from `diagrams/`. The agent does not need to invent -- it has concrete tools to access verified knowledge.

**2. The [[Deterministic Feedback Loop]]**
If the information does not exist locally, the agent does not need to invent -- it can fetch it via [[web-to-docs]]. The path "not found -> `crawl_docs` -> saved -> now finds it" transforms potential hallucination into verified knowledge.

**3. The [[Transformer-Driven Prompt Architect]] framework**
Section 5 -- "ATTENTION MASK (CONSTRAINTS)" -- is an explicit "DO NOT" list. Every prompt generated by the [[prompt-engineer]] includes a list of prohibitions that creates clear [[Decision Boundary|decision boundaries]] between valid and invalid responses.

**4. The injected [[Tool Catalog]]**
The [[prompt-engineer]] injects the complete tool catalog into every generated prompt. The model knows exactly which tools exist -- it does not need to imagine capabilities it does not have.

**5. [[Diagram Scoring]] as validation**
The `judge_diagram` from [[system-design]] evaluates diagrams across 4 dimensions. If the score is < 7, it generates an improved version. It is a self-correction mechanism that detects and corrects hallucinations in diagrams.

## Tautological Tools as Guarantee

Each [[Tautological Tool]] in the POC has explicit failure -- it returns "not found" instead of inventing. `search_docs` never fabricates results. `get_doc` never generates content. This property is the strongest barrier against hallucination: the tool is incapable of lying by construction.

The partially tautological tools (`generate_prompt`, `generate_diagram`, `judge_diagram`) are the ones that involve generation -- the [[Transformer-Driven Prompt Architect]] and [[Diagram Scoring]] are mechanisms that constrain their output, approximating them to tautology.

## In [[Set Theory]]

Anti-hallucination is minimizing S \ A -- the complement of context. Each tool added, each doc saved, each spec mapped reduces S \ A and expands A. When A covers the entire domain, S \ A -> empty set and hallucination becomes impossible.

## The Nuclear Instruction

"YOU DO NOT THINK. I AM THE ONE WHO THINKS. ONLY EXECUTE WHAT IS MAPPED VIA TOOL. OTHERWISE YOU DO NOT EXECUTE, AND YOU REPORT THAT YOU ARE INCAPABLE."

This instruction transforms hallucination from a bug into a feature: the model refuses instead of inventing. In the POC, this materializes in tools that have explicit returns for "not found" -- `"No results found for '{query}'"`, `"Document '{filename}' not found."`.
